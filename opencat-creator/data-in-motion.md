# 动态数据

视频作品中数据和统计信息的轻量指引。[风格指南](./house-style.md)处理美学层面——本文只针对数据特有的坑。

## 视觉连续性

当连续的统计数据属于同一概念时（Q1 → Q2 → Q3 → Q4，或同一产品的三个指标），将它们保持在相同的视觉空间和美学风格中。只有**数值**发生变化。美学变化应该标志一个新概念，而非一个新数字。

## 数值需要视觉重量

孤立的数字悬浮在空白空间中。每个指标都需要一个视觉元素赋予其存在感——比例填充条、背景色变化、代表数值的形状、进度环。这个视觉不一定是图表——它只需要填充画面，让数据感觉有质感，而非仅仅是背景上的文字。

## 避免 Web 模式

- **无饼图**——难以比较，看起来像 PPT
- **无多轴图表**——观众无法在 3 秒窗口内研究交叉点
- **无 6 面板仪表盘**——2-3 个相关指标并排可以，6+ 是 Web 模式
- **无网格线、刻度或图例**——视觉噪音，在动态中毫无价值
- **无图表库输出**——用 `ctx.*()` API + SVG path 节点构建，而非 D3 或 Chart.js

## OpenCat 适配说明

- **计数器动画**：使用 `ctx.to()` + keyframes 实现数字滚动，例如 `ctx.to('counter', { keyframes: { opacity: [0, 1] }, duration: 60 })`
- **填充条动画**：使用 `ctx.fromTo()` 驱动进度条/比例条的宽度或路径长度变化，例如 `ctx.fromTo('bar', { scaleX: 0 }, { scaleX: 1, duration: 45 })`
- **数据源**：所有数值定义在 JSONL 格式中，通过 `references/` 前缀引用资产路径
- **时序编排**：利用 keyframes 的 offset 属性控制多数据点的入场节奏
