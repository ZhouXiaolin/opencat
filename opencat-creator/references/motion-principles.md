# 动效原则

## 护栏规则

这些规则你明明知道但总违反。打住。

- **不要在每条补间上用同一个缓动函数。** 你默认所有动画都用 `ease-out`。像换字体字重一样换缓动——一个场景中不超过 2 条独立补间用同一个缓动。
- **不要在每段动画上用同一个速度。** 你默认所有动画都用 12-15 帧。最慢的场景应该比最快的慢 3 倍。刻意变化时长。
- **不要让所有元素从同一个方向入场。** 你默认每个元素都用 `y: 30, opacity: 0`。变化：从左边、从右边、从缩放、纯不透明度、字间距。
- **不要在每个场景用同一个交错节奏。** 每个场景需要自己的韵律。
- **不要在每个场景都用环境缩放。** 每个场景选不同的环境动效：慢速平移、微妙旋转、缩放推进、色彩偏移、或者不动。动后的静止很有力量。
- **不要在 t=0 开始。** 将首个动画偏移 3-9 帧。零延迟感觉像跳切。

## 未经要求不要做的事

### 缓动是情感，不是技术

过渡是动词，缓动是副词。用 `ease-out`（原 expo.out）入场 = 自信。用 `ease-in-out`（原 sine.inOut）= 梦幻。用 `elastic-out`（原 elastic.out）= 俏皮。同一种运动，不同的含义。刻意选择副词。

**方向规则——不可选项：**

- `ease-out`（结尾减速）用于元素入场。开始快，减速。感觉响应迅速。这是你的默认。
- `ease-in`（开头加速）用于元素退场。开始慢，加速离开。把它们甩出去。
- `ease-in-out`（两头慢）用于元素在位置之间移动。

你一直搞反。入场用 `ease-in` 感觉拖沓。退场用 `ease-out` 感觉不情愿。

### 速度传达重量

- 快速（4-9 帧）— 能量、紧迫、自信
- 中速（9-15 帧）— 专业、大多数内容
- 慢速（15-24 帧）— 重量感、奢华、沉思
- 极慢（24-60 帧）— 电影感、情感、氛围

### 场景结构：建场 / 呼吸 / 收场

每个场景有三个阶段。你总把所有东西堆在建场，不给呼吸和收场留空间。

- **建场（0-30%）** — 元素交错入场。不要一次全倾倒。
- **呼吸（30-70%）** — 内容可见，由一个环境动效赋予生命力。
- **收场（70-100%）** — 退场或决定性结束。退场比入场快。

### 过渡即意义

- **交叉淡入淡出** = "这还在继续"
- **硬切** = "醒醒" / 中断
- **慢速溶解** = "跟我漂移"

你所有地方都用交叉淡入淡出。在需要中断和场景切换时用硬切。

### 编排即层级

最先运动的元素被认为最重要。按重要性顺序交错，而非 DOM 顺序。不要等上一个完成——重叠入场。总交错序列不超过 500ms（15 帧），无论元素数量。

### 非对称

入场比退场长。一个卡片用 12 帧出现但用 7-8 帧消失。

## 视觉构图

你为网页构建。视频帧不是页面。

- **每场景最少两个焦点。** 视线需要移动的路径。永远不要单个文本块漂浮在空白空间中。
- **填满画面。** Hero 文字：画布宽度的 60-80%。你总想用网页尺寸的元素。别。
- **每场景最少三层。** 背景处理（发光、超大褪色文字、色块）。前景内容。强调元素（分割线、标签、数据条）。
- **背景不是空的。** 径向发光、超大褪色文字溢出画布、微妙边框面板、极细线条。纯色 #000 读作"什么都没加载"。
- **锚定到边缘。** 将内容固定在左/上或右/下。居中和漂浮是网页布局模式。
- **分割画面。** 左数据面板，右内容。顶部栏加信息，下方全宽。基于区域的布局，不是居中堆叠。
- **使用结构元素。** 分割线、分隔器、边框面板。它们创造视线路径且动效效果好（scaleX: 0 → 1）。

## 图片动效处理

永远不要嵌入一张扁平静态图片。每张图片必须有动效处理：

- **透视倾斜**：使用 `ctx.set('hero', { rotate: -2, scale: 0.96 })` 模拟轻微倾斜——创造深度。复杂 3D 透视需通过 `type: "canvas"` + CanvasKit 实现。
- **慢速缩放（Ken Burns）**：使用 `ctx.to('hero', { scale: 1.04, duration: 帧数, ease: "none" })` —— 让照片有电影感
- **设备边框**：用 Tailwind `rounded-*` 和 `shadow-*` 包裹在笔记本/手机形状中
- **浮动 UI**：提取关键元素，在不同的 z 深度上单独动画制造视差
- **滚动揭示**：将图片裁剪到视口窗口，动画其 `y` 位置

## 核心规则

以下规则源自两个独立的 website-to-hyperframes 构建（2026-04-20）。当时 composition lint 干净但仍然出问题——元素从不出现、环境动效不能拖拽、入场补间无声杀死目标。Linter 抓不到这些；作者必须遵守这些规则。

### 禁止在同一元素上堆叠两个变换补间

常见错误：对同一个元素做 `y` 入场加 `scale` Ken Burns。第二个补间在构造时写入元素的初始状态，覆盖第一个补间设置的状态——使元素不可见或偏离画面，且没有 lint 警告。两种修复方式：

```jsonl
<!-- 错误：一个元素两个变换 -->
{"id":"hero","parentId":"scene1","type":"image","className":"w-[400px] h-[300px] object-cover","path":"hero.jpg"}
{"type":"script","parentId":"scene1","src":"ctx.timeline()\n  .fromTo('hero', { y: 50, opacity: 0, scale: 1.0 }, { y: 0, opacity: 1, scale: 1, duration: 18, ease: 'ease-out' }, 0)\n  .to('hero', { scale: 1.04, duration: beat }, 0); // 杀死了入场"}

<!-- 正确选项 A：合并为一个 fromTo -->
{"type":"script","parentId":"scene1","src":"ctx.timeline().fromTo(\n  'hero',\n  { y: 50, opacity: 0, scale: 1.0 },\n  { y: 0, opacity: 1, scale: 1.04, duration: beat, ease: 'none' },\n  0\n);"}

<!-- 正确选项 B：拆分到父元素 + 子元素 -->
{"id":"hero-wrap","parentId":"scene1","type":"div","className":"w-[400px] h-[300px]"}
{"id":"hero","parentId":"hero-wrap","type":"image","className":"w-full h-full object-cover","path":"hero.jpg"}
{"type":"script","parentId":"scene1","src":"ctx.timeline()\n  .fromTo('hero-wrap', { y: 50, opacity: 0 }, { y: 0, opacity: 1, duration: 18, ease: 'ease-out' }, 0)\n  .to('hero', { scale: 1.04, duration: beat }, 0);"}
```

### 场景内优先使用 `fromTo` 而非 `to`

`ctx.to()` 从当前状态开始动画。在非线性 seek（渲染引擎会做）中，当前状态可能是上一次 seek 的残留。显式的 `ctx.fromTo()` 定义了起始帧和结束帧的状态，使每个时间点的状态是确定的：

```js
// 脆弱：取决于当前状态
ctx.to('el', { opacity: 1, y: 0, duration: 18, ease: "ease-out" });

// 确定：两端状态都显式定义
ctx.fromTo('el', { opacity: 0, y: 50 }, { opacity: 1, y: 0, duration: 18, ease: "ease-out" });
```

### 环境动效必须挂载到时间线上

光晕、微光、轻柔浮动循环、logo 呼吸——所有这些都必须添加到场景的时间线上，不能单独调用 `ctx.to()`。单独的补间在非线性 seek 中不会正确拖拽，导致渲染视频中效果消失，即使预览看起来正确：

```js
// 错误：游离在时间线之外，渲染中不工作
ctx.to("aura", { scale: 1.08, yoyo: true, repeat: 5, duration: 36 });

// 正确：挂载到时间线，确定性，可渲染
ctx.timeline().to("aura", { scale: 1.08, yoyo: true, repeat: 5, duration: 36 }, 0);
```

### 每个场景边界硬杀死

任何在节拍边界改变可见性的元素，在其淡出后都需要一个确定性的 `ctx.set()` 杀死。因为后续对该元素的补间（或兄弟补间的状态残留）可能将其复活。应用于每个有退场动画的元素：

```js
ctx.fromTo('el', { opacity: 1 }, { opacity: 0, duration: 9, ease: "ease-in" }, beatEnd);
ctx.set('el', { opacity: 0 }, beatEnd + 9); // 确定性杀死
```

以上是确切的规则和代码示例——不要概括或缩短。它们存在是因为 composition 即使 lint 干净，没有它们仍然出问题。
