# 旁白与脚本

如何为视频合成撰写旁白脚本。当合成中包含画外音或 TTS 时阅读。

## 节奏

- **每秒 2.5 词**是自然的语速
- 15 秒 ≈ 37 词，30 秒 ≈ 75 词，60 秒 ≈ 150 词
- 为停顿留出空间。句与句之间的留白是特性，不是冷场
- 脚本应当感觉比视频**更短**——视觉呼吸空间至关重要

## 语气

像人一样写作，不是宣传册：

- 使用自然的缩写："它能"、"那就是"、"我们已经"
- 变换句子长短——短促有力的短句与流畅的长句交替
- 大声读出来。如果听起来像机器，重写
- 除非目标受众期待专业术语，否则避免使用

## 数字发音

写下你希望声音说出的内容。TTS 逐字朗读。

| 产品中显示 | 脚本中应写作           |
| ---------- | ---------------------- |
| 135+       | 超过一百三十五         |
| $1.9T      | 近两万亿美元           |
| 99.999%    | 百分之九十九点九九九   |
| 200M+      | 超过两亿               |
| 10x        | 十倍                   |
| API        | A P I                  |
| opencat.ai | opencat 点 ai          |

画面可以显示精确数字，而声音将其取整。

## 结构

产品视频的结构：

1. **钩子**——这个产品有什么令人惊讶或印象深刻的地方？一个大胆的断言、一个挑衅性的问题、一个对比，或一个引人注目的数字。这是开场白。**变化钩子的类型**——不要每次都默认用数字统计。
2. **故事**——产品做什么？谁在使用它？保持具体。
3. **证明**——数据、客户名称、社交证明。来自产品的真实数据。
4. **行动号召**——观众应该做什么？

并非每个视频都需要全部四部分。15 秒的社交广告可能只需要钩子 + 证明 + 行动号召。60 秒的产品展示则全部使用，故事部分更长。

## 开场白

视频中最重要的句子。必须在最初 3 秒内制造紧张、好奇或惊喜。

有效的模式：

- **大胆的断言**："支撑互联网经济的金融基础设施。"
- **挑衅性的问题**："如果你的数据库能思考会怎样？"
- **对比**："你的 AI 代理已经知道如何制作视频。它只需要正确的格式。"
- **惊人的数字**："近两万亿美元。"（谨慎使用——不是每个视频都应以数据开头）

如果开场白很通用（"欢迎使用我们的产品" / "让我介绍一下"），请重新开始。

## 示例

取自一个 62 秒产品发布视频（团队参考）：

```
你的 AI 代理已经知道如何制作视频。
它只需要正确的格式。

这就是 OpenCat。一个 JSONL 运动图形引擎。声明节点，输出动画。

一个节点是一个图层。变换属性是你的动画轨道。
Skia 是你的渲染器。QuickJS 是你的脚本引擎。

任何能用坐标和关键帧描述的运动都可以是动画的一帧。

节点树。变换。不透明度。缩放。旋转。缓动函数。

放入文字、图片、形状——它们都能组合在一起。

不需要代理学习新的领域特定语言。
只需要 JSONL。

代理编写它。运行时逐帧渲染为视频。
它是确定性的。每次输出完全相同。

告诉 CLI 你要做什么。
看它构建。

OpenCat。去创造点什么。
```

注：约 140 词对应 62 秒的语速约为 2.3 词/秒，为停顿和视觉呼吸留出空间。
