# 排版

OpenCat 使用 Tailwind 字体体系。字体通过 `className` 中的 Tailwind token 控制。

## 禁用列表

训练数据默认项——每个 LLM 都会抓取的字体。这些造成 composition 间的单一化。

Inter、Roboto、Open Sans、Noto Sans、Arimo、Lato、Source Sans、PT Sans、Nunito、Poppins、Outfit、Sora、Playfair Display、Cormorant Garamond、Bodoni Moda、EB Garamond、Cinzel、Prata、Syne

**Syne 尤其**是最被过度使用的"独特"展示字体。它是 AI 设计的即时标志。

## 护栏

这些规则你知道但总违反。打住。

- **不要配对两个无衬线。** 你一直这样做——一个标题、一个正文。跨过边界：衬线 + 无衬线，或无衬线 + 等宽。
- **每个场景一个表现力字体。** 你选两个有趣的字体试图让它"更好"。一个表演，一个退后。
- **字重对比必须极端。** 你默认 400 vs 700。视频需要 300 vs 900。差异必须在运动中一目了然。
- **视频尺寸，不是网页尺寸。** 正文：`text-[20px]` 起。标题：`text-[60px]+`。数据标签：`text-[16px]`。你会想用 14px。不要。

## 未经要求不要做的事

- **张力应该有意义。** 不要模式匹配配对。问为什么这两种字体不协调。配对应该体现内容的矛盾——机械 vs 人文、公共 vs 私人、机构 vs 个人。如果你无法阐述张力，那就是任意的。
- **语域切换。** 为不同的交流模式分配不同的字体——一个声音用于陈述，另一个用于数据，另一个用于引用。不是页面上的层级。是对话中的声音。
- **张力可以存在于单一字体内部。** 一个看起来熟悉但秘密奇怪的字体，与观众的期望产生张力，而不是与另一种字体。
- **一个变量改变 = 戏剧性对比。** 相同字形，等宽 vs 比例。相同家族在不同光学尺寸下。只改变节奏而其他一切保持不变。
- **双重人格有效。** 两种表现力字体可以共存，如果它们共享一种态度（都离经叛道、都精确）即使形式完全不同。
- **时间即层级。** 第一个出现的元素是最重要的。在视频中，序列取代位置。
- **动效即排版。** 一个词如何进入承载着与字体一样多的含义。0.1 秒的猛击 vs 2 秒的淡入——同一字体，完全不同的信息。
- **固定阅读时间。** 屏幕上 3 秒 = 必须在 2 秒内可读。更少的字，更大的字号。
- **字间距比网页更紧。** 展示尺寸用 -0.03em 到 -0.05em。视频编码压缩字母细节。

## 暗背景调整

亮色文字在暗色背景上产生两种光学错觉，需要补偿：

- **视觉字重增加。** 亮-on-暗比暗-on-亮在同一 `font-weight` 下读起来更重。正文用 350 而非 400。标题受影响较小因为尺寸补偿。
- **视觉间距更紧。** 字形周围的光晕减少感知间隙。将 `line-height` 增加 0.05-0.1。展示尺寸增加 0.01em `letter-spacing` 来抵消。

## OpenType 特性（数据场景）

大多数字体附带默认关闭的 OpenType 特性。数据场景下开启：

```jsonl
{"id":"stat-value","parentId":"scene1","type":"text","className":"text-[36px] font-bold tabular-nums","text":"1,234"}
```

Tailwind token：
- `tabular-nums` — 数字在列中垂直对齐（`font-variant-numeric: tabular-nums`）
- `diagonal-fractions` — 将 1/2 渲染为 ½
- `lining-nums` — 统一高度的数字
- `oldstyle-nums` — 传统风格数字

`tabular-nums` 在数字垂直堆叠时必不可少——统计标注、计时器、记分板、数据表。没有它，数字有比例宽度，列不对齐。

## 选择思维

不要按类别反射选字体（社论 → 衬线，技术 → 等宽，现代 → 几何无衬线）。那是模式匹配，不是设计。

1. **命名语域。** 内容以什么声音说话？机构权威？个人忏悔？技术精确？随意离经叛道？语域比类别更缩窄范围。
2. **物理想象。** 想象字体作为品牌可以交付的物理对象——博物馆展览说明牌、手绘商店招牌、1970 年代大型机终端手册、外套内部的面料标签、印刷在廉价新闻纸上的童书、税表。哪个物理对象符合语域，就指向正确**类型**的字体。
3. **拒绝第一直觉。** 感觉对的字体通常是该语域的训练数据默认。如果你上次也选了它，找别的。
4. **交叉验证假设。** 社论 brief 不一定需要衬线。技术 brief 不一定需要无衬线。儿童产品不一定需要圆润展示字体。最独特的选择常常违反类别预期。
