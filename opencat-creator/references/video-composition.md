# Video Composition

视频帧不是网页。这些规则适用于每个 composition，无论品牌、风格或 design.md。

## design.md 是品牌，不是布局

design.md 定义品牌的外观：颜色、情绪、约束。它不定义如何构图视频帧。在视频合适的强度级别使用品牌色 — 不是在网页 UI 透明度层级。

**严格来自 design.md：** Tailwind token（包括背景色）、字重关系、Do's and Don'ts。如果用户选择了亮色画布，使用亮色。选择了暗色，使用暗色。不要覆盖他的色板。

**适配视频：** 字号、间距、装饰透明度、边框重量。网页 UI 卡片的 `border-[1px] border-slate-200` 在视频中看不见。品牌色是神圣的；应用方式是你的。

## Density

一个场景有 3 个元素看起来空。8-10 个感觉有生命力。

每个场景需要：

- **背景纹理** — 径向发光、超大幽灵文字、色块、杂色、网格。从不纯色平铺。
- **中景内容** — 实际要传达的信息。卡片、数据、代码块、图片。
- **前景强调** — 分割线、标签、数据条、刻度标记。让它感觉被制作过。

目标是每场景 8-10 个视觉元素。其中两个应该是用户没要求的装饰元素 — 你添加它们因为空画面看起来坏了。

## Color Presence

柔和没问题。平淡不行。每个场景应至少有一个拉眼球的颜色。

- 品牌强调色应该**可见** — 不是 5% 不透明度的发光在压缩中丢失。15-25% 用于氛围，纯饱和用于焦点元素。
- **亮色画布与暗色不同。** 暗色上：强调发光自然弹出。亮色上：使用更粗的边框（2px+ 实体），更强的结构元素（分割线、分隔线），和全饱和强调色。亮背景需要纹理（微妙杂色、图案）以避免"空白幻灯片"感。不要切换为暗色 — 让亮色有电影感。
- 中性色向品牌色相偏移。死灰色读起来像未经设计。

## Scale

网页尺寸在视频中不可见。一切放大。

| 元素 | 网页 | 视频 |
|------|------|------|
| 标题 | 16-20px text | `text-[60px]-[80px]` |
| 正文 | 14-16px | `text-[24px]-[36px]` |
| 标签 | 12px | `text-[16px]-[20px]` |
| 装饰不透明度 | 3-8% | 12-25% |
| 边框 | `border`（1px） | `border-2` 或 `border-4` |
| 内边距 | 16-32px | `p-[60px]-[140px]` |

如果你在 className 中写小于 `text-[20px]` 的字号，给出理由。如果装饰不透明度低于 10%，它不可见。

## Motion Intensity

微妙在 30fps 下读起来像静止。宁可动得比感觉安全的多。

- 每个装饰元素应有环境动效：breathe、drift、pulse、orbit。静态装饰感觉死板。
- 每个场景变化动效模式 — 不要重复相同的环境模式。
- 场景入场应使用 3 种以上不同的 easing 和方向。如果每个元素都从 `{ opacity: 0, y: 30 }` 入场，这个场景没有编排。

## Frame Composition

- **最少两个焦点。** 视线需要移动的路径。
- **填满画面。** Hero 文字：画布宽度的 60-80%。
- **锚定到边缘。** 将内容固定在左/上或右/下。居中和漂浮是网页布局模式。
- **分割画面。** 左数据面板，右内容。顶部栏+信息，下方全宽。基于区域的布局优于居中堆叠。
- **使用结构元素。** 分割线、分隔器、边框面板。它们创造视线路径并且动效效果好（`scaleX: 0` → `1`）。
