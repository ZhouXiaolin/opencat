# 视频构图

视频帧不是网页。在滚动页面上有效的网页布局模式在固定帧合成中经常显得破碎。

---

## 基本原则

### 两个焦点 / 场景

眼睛需要一个去处。一个浮动在空白的单个文本框读起来像未完成。

### 填满画面

- Hero 文字通常需要画布宽度的 **60-80%**
- 网页字号（16px body、32px 标题）在视频距离消失
- 视频字号：标题 60-80px，正文 24-36px，标签 16-20px

### 三层结构 / 场景

1. **背景处理** — 径向发光、超大褪色文字、色块
2. **前景内容** — 标题、数据、图片、视频
3. **强调元素** — 分割线、标签、数据条、装饰

一个只有一层的场景读起来扁平。

### 背景不是空的

- 径向发光（强调色着色，低透明度，呼吸缩放）
- 幽灵文字（主题词 3-8% 透明度，非常大的字号，缓慢漂浮）
- 强调线（细水平线 `border-t w-full`，微弱脉冲动画）
- 杂色/噪点叠加、几何形状、网格图案
- 主题装饰（轨道环、唱片纹、数据网格线）

纯色 `#000` 读作"什么都没有加载"。

### 锚定到边缘

- 内容固定到左/上或右/下
- 居中浮动是网页模式，在 16:9 画布上显得失落

### 分割画面

- 左数据面板 + 右内容
- 顶部元数据栏 + 下面全宽内容
- 基于区域的布局 beats 居中堆叠

### 使用结构元素

- 分割线、边框面板创造视线路径
- 它们也动画很好（`scaleX` 从 0）

## 图片动效处理

嵌入的图片不应放平 — 每张图片都应获得某种动效处理：

- **透视倾斜** — 轻微 `rotate` + `scale` 创造深度
- **慢速缩放（Ken Burns）** — `scale: 1` → `1.04` 给场景时长。让照片感觉电影感而不是贴上去的
- **设备边框** — 用 `rounded-*` 和 `shadow-*` 包裹
- **浮动 UI** — 提取关键元素在不同 z 深度单独动画制造视差
- **滚动揭示** — 裁剪图片到视口窗口，动画其 `y` 位置

## 排版对比度

- 亮背景：正文 `#000000` + 强调色用于 CTA
- 暗背景：正文 `#FFFFFF` + 强调色用于 CTA
- 中性色向强调色偏移 — 即使是微弱的 warmth/coolness 也比死灰色好
- WCAG AA 对比度（4.5:1 正文、3:1 大文字）是基准
