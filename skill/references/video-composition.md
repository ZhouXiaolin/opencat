# 视频构图

视频画面不是网页。在滚动页面上有效的网页布局模式在固定画面合成中经常显得破碎。

---

## design.md 是品牌，不是布局

`design.md` 定义品牌看起来是什么：颜色、字体、性格、约束。它不定义视频画面怎么构图。

必须严格遵守：

- hex 色值
- 字体族和字重关系
- 明暗方向
- Do's and Don'ts

可以为视频媒介调整：

- 字号
- 间距
- 装饰透明度
- 边框粗细
- 卡片/面板处理

网页 UI 里的 1px 边框、6% 阴影、14px 正文在视频里通常不可见。品牌色是事实，视频尺度由你负责。

## 基本原则

### 密度

每个场景目标 8-10 个视觉元素。不是都要抢主视觉，而是形成完整层次：

- **背景纹理**：径向光、幽灵文字、网格、噪点、色块、轨道线
- **中景内容**：标题、数据、UI、图像、代码块、卡片
- **前景强调**：分割线、标签、刻度、坐标、logo、进度条、光标

一个只有 3 个元素的 beat 通常会显得空。至少 2 个装饰元素应由设计主动补足，不必等用户要求。

### 色彩存在感

柔和可以，扁平不行。每个场景至少有一个能抓住视线的颜色。

- 品牌强调色必须可见：氛围层 15-25% 透明度，焦点元素可用全饱和
- 暗背景适合 glow，但避免整屏线性渐变导致压缩 banding
- 亮背景需要更强结构：2px+ 边框、实色强调、纹理和分割线
- 中性色向品牌色偏移，避免死灰

### 两个焦点 / 场景

眼睛需要一个去处。一个浮动在空白的单个文本框读起来像未完成。

### 填满画面

- Hero 文字通常需要画布宽度的 **60-80%**
- 网页字号（16px body、32px 标题）在视频距离消失
- 视频字号：标题 64-120px，正文 28-42px，标签 18-24px
- 装饰透明度：12-25% 才能稳定进入编码后画面
- 边框：2-4px 比 1px 更适合视频
- 场景 padding：60-140px 是常见范围，取决于画幅和密度

写低于 24px 的文字时必须有理由；低于 10% 的装饰透明度通常等于不存在。

### 三层结构 / 场景

1. **背景处理** — 径向发光、超大褪色文字、色块
2. **前景内容** — 标题、数据、图片、视频
3. **强调元素** — 分割线、标签、数据条、装饰

一个只有一层的场景读起来扁平。

### 背景不是空的

- 径向发光（强调色着色，低透明度，呼吸缩放）
- 幽灵文字（主题词 3-8% 透明度，非常大的字号，缓慢漂浮）
- 强调线（细水平线 `border-t w-full`，微弱脉冲动画）
- 杂色/噪点叠加、几何形状、网格图案
- 主题装饰（轨道环、唱片纹、数据网格线）

纯色 `#000` 读作"什么都没有加载"。

### 锚定到边缘

- 内容固定到左/上或右/下
- 居中浮动是网页模式，在 16:9 画布上显得失落

### 分割画面

- 左数据面板 + 右内容
- 顶部元数据栏 + 下面全宽内容
- 基于区域的布局 beats 居中堆叠

### 使用结构元素

- 分割线、边框面板创造视线路径
- 它们也动画很好（`scaleX` 从 0）

## 图片动效处理

嵌入的图片不应放平 — 每张图片都应获得某种动效处理：

- **透视倾斜** — 轻微 `rotate` + `scale` 创造深度
- **慢速缩放（Ken Burns）** — `scale: 1` → `1.04` 给场景时长。让照片感觉电影感而不是贴上去的
- **设备边框** — 用 `rounded-*` 和 `shadow-*` 包裹
- **浮动 UI** — 提取关键元素在不同 z 深度单独动画制造视差
- **滚动揭示** — 裁剪图片到视口窗口，动画其 `y` 位置

## 排版对比度

- 亮背景：正文 `#000000` + 强调色用于 CTA
- 暗背景：正文 `#FFFFFF` + 强调色用于 CTA
- 中性色向强调色偏移 — 即使是微弱的 warmth/coolness 也比死灰色好
- WCAG AA 对比度（4.5:1 正文、3:1 大文字）是基准

## 运动强度

30fps 视频里“微妙”容易读作静止：

- 每个装饰元素应有环境动效：drift、pulse、orbit、breathe、color shift
- 不同场景不要重复同一种 ambient zoom
- 入场至少混合 3 种 ease / 方向 / duration
- 结构线和分割线适合 `scaleX` / `clip` / `wipe` 动画
